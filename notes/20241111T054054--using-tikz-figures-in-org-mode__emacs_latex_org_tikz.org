* Option 1

Include LaTeX/Tikz as a source block. Useful if exporting to formats
besides PDF.

#+header: :exports results :results file raw :file (org-babel-temp-file "tikz-" ".png")
#+header: :imagemagick t :iminoptions -density 600 :imoutoptions -geometry 400 -flatten :fit t
#+header: :headers '("\\usepackage{tikz}")
#+begin_src latex
\begin{tikzpicture}
    \node [circle, draw, fill=red!20] at (0,0) {1}
    child { node [circle, draw, fill=blue!30] {2}
      child { node [circle, draw, fill=green!30] {3} }
      child { node [circle, draw, fill=yellow!30] {4} }};
\end{tikzpicture}
#+end_src

* Option 2

Add LaTeX preview support for Tikz figures.

#+begin_src emacs-lisp
(with-eval-after-load 'org
    (add-to-list 'org-latex-packages-alist '("" "tikz" t)))
#+end_src

#+RESULTS:
|   | tikz | t |

That alone might suffice, but dvipng doesn't behave nicely, so use
imagemagick instead.

#+begin_src emacs-lisp
(setq org-preview-latex-default-process 'imagemagick)
#+end_src

#+RESULTS:
: imagemagick

Now give this block the usual =C-c C-x C-l=.

\begin{tikzpicture}
  \node [circle, draw, fill=red!50] at (0,0) {1}
  child { node [circle, draw, fill=blue!50] {2}
    child { node [circle, draw, fill=green!30] {3} }
    child { node [circle, draw, fill=yellow!30] {4} }};
\end{tikzpicture}

Naturally, best to add the above to =init.el=.
