Preparing to write the SIVE paper, I decided to gather some results. I
sent a 1 Hz pulse from a clock-authority/audio-server Teensy to a
network of three further subscriber Teensies with the following IPs

- 192.168.48.191
- 192.168.48.203
- 192.168.48.234

I attached probes to pin 7 (I2S data) of the authority and one
subscriber at a time, and measured the offset of the rising edge of
the 1 Hz pulse, writing that offset at each triggered acquisition of
the authority to a CSV file via a script in the /Waveforms/
application.

#+begin_src js :noeval
var index = 0;

Scope.Channel2.data.forEach(function(element, idx){
    // Find the index of the datapoint where channel 2 first exceeds the trigger value
    if(index == 0 && element > Scope.Trigger.Level.value) {
        index = idx
    }
})

// Find the time associated with the index found above.
const time = Scope.Channel2.TimeOfIndex(index)

// Append the offset time found to a CSV file.
FileAppendLine("~/org/papers/sive-2025/data/191-ptp.csv", [ // e.g.
    time
])
#+end_src

I took just over 600 measurements (10 mins' worth) for each pair of IP
and test condition:
- Without PTP.
  + To clarify, PTP running, but no adjustment applied to subscriber
    audio clocks.
- With PTP, but simulating an unmanaged switch by deactivating the PTP
  profile on the switch.
- With PTP.

Finally, I combined these into one large CSV:

#+begin_src julia :noeval
using CSV, DataFrames

t41lastBytes = [191, 203, 234]
df = DataFrame()
for b in t41lastBytes
    for c in ["noptp", "unmanaged", "ptp"]
        tmp = CSV.read("./data/$b-$c.csv", DataFrame; header=false).Column1
        tmp = vcat(tmp, missings(610 - length(tmp)))
        df[!, "$b-$c"] = tmp
    end
end
CSV.write("./combined-data.csv", df, separator=',', header=true)
#+end_src

I gave the resulting file the =denote-rename-file= treatment.

* Visualising the data

First, load some Julia packages

#+begin_src julia :results none
using CSV, DataFrames, Plots
#+end_src

Now load the data. I recorded just over 600 samples in each case; take
just the first 600.

#+begin_src julia :results none
data = CSV.read(
    "./20241218T154907--teensy-ptp-drift-measurements-data__data_ptp_teensy.csv", DataFrame;
    limit=600
)
#+end_src

This function will take a category --- "noptp", "unmanaged", or "ptp"
--- and generate the corresponding plot:

#+begin_src julia :results none
function plotDrift(dataCategory, plotFilepath)
    t41lastBytes = [191, 203, 234]
    plot(legendtitle="IP address", yformatter=:scientific)
    xlabel!("Time (s)")
    ylabel!("Offset (s)");
    for b in t41lastBytes
        column = data[!, "$b-$dataCategory"]
        # Remove initial offset
        column .= column .- column[1]
        plot!(column, label="...48.$b", linewidth=1.5)
    end

    savefig(plotFilepath)
end
#+end_src

As one would expect, we see more-or-less linear drift, with two
subscribers running fast (one, =...48.234=, significantly faster than
the other, =...48.203=), and one slow. =...48.191= gets far enough
behind that its buffer read index wraps around. Here, for =...48.234=,
we see drift on the order of...

#+begin_src emacs-lisp
(format "%f ppm" (* (/ -4e-3 600) 1e6)) 
#+end_src

#+RESULTS:
: -6.666667 ppm

Which is just about commensurate with what we'd expect to see for a
commercial grade XO.

** With PTP

plotDrift("ptp", "$dir/ptp.png")

There's still some drift, but now we're talking! In the worst case,
that of =...48.234=, we have:

#+begin_src emacs-lisp
(format "%f ppb" (* (/ 1e-6 600) 1e9)) 
#+end_src

#+RESULTS:
: 1.666667 ppb

...or \(1\frac{2}{3}\) ns/s; 1 \micro{}s in ten minutes.

There's still some drift, however. Interestingly, it appears to
/mirror/ the case from the first plot, with =191= running slightly
fast (rather than slightly slow), =203= running slow, and =234=
running about 2.5x as slow as =203=. This suggests that there's some
value, derivable from the /raw drift/, that can be applied in inverse
fashion to correct the remaining drift.
