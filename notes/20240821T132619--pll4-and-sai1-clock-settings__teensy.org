  + Not clear, at the time of writing, how these fields differ;

The output of the above is the input for the following, which

[[./20240821T133618--teensy-clock-setup-sai1__teensy.png]]

Pictured above are the registers and fields used by Teensy. The ones
of interest are:

- Audio PLL
  - ~DIV~: 28
  - ~NUM~: 2240
  - ~DENOM~: 10000
- ~SAI1_CLK_PRED~: 4
- ~SAI1_CLK_PODF~: 15

Plus maybe:
- ~PLL4_POST_DIV~: 1
- ~AUDIO_DIV~: 1

** Audio Clock Generation

Let's assume that the clock source isn't bypassed, and that
~PLL_BYPASS = 0~ and ~PLL4_ENABLE = 1~. 

- The 24 MHz clock source (OSC 24M) is the input to Audio PLL;
- Audio PLL multiplies its input by ~DIV + NUM/DENOM~
  + In this case we have:
#+begin_src julia :session :results output :exports both :eval never-export
OSC_24M = 24e6
DIV = 28
NUM = 2240
DENOM = 10000
PLL4 = OSC_24M * (DIV + NUM/DENOM)
@printf "PLL4 Frequency: %s Hz" cfmt("%'d", PLL4)
#+end_src

#+RESULTS:
: PLL4 Frequency: 677,376,000 Hz

- ~POST_DIV_SELECT~ and ~AUDIO_DIV~ are applied, but these both take a
  value of 1 in this case, so no change.
- The synchronous audio interface, SAI1, receives the PLL4 clock and
  divides it, first by ~SAI1_CLK_PRED~, then by ~SAI1_CLK_PODF~, and
  produces ~SAI1_CLK_ROOT~:
#+begin_src julia :session :results output :exports both :eval never-export
SAI1_CLK_PRED = 4
SAI1_CLK_PRED_FREQ = PLL4 / SAI1_CLK_PRED
SAI1_CLK_PODF = 15
SAI1_CLK_ROOT = SAI1_CLK_PRED_FREQ / 15
@printf "SAI1_CLK_ROOT Frequency: %s Hz" cfmt("%'d", SAI1_CLK_ROOT)
#+end_src

#+RESULTS:
: SAI1_CLK_ROOT Frequency: 11,289,600 Hz
Which is \(2^{8}\) times the target sampling rate:
#+begin_src julia :session :results output :exports both :eval never-export
audioSampleRate = SAI1_CLK_ROOT / 1<<8
@printf "Audio sample rate: %s Hz" cfmt("%'d", audioSampleRate)
#+end_src

#+RESULTS:
: Audio sample rate: 44,100 Hz

* Audio clock capabilities

Let's set off in reverse; ~SAI1_CLK_ROOT~ (\(F_{I}\)) should equal 256
times the target sample rate (\(F_{s}\)):[fn:1]

#+NAME: eq:sai1-freq1
\begin{equation}
F_{I} = 2^{8}\cdot{}F_{s}
\end{equation}

Config Tools says \(F_{I}\) must be lower than or equal to 66 MHz, so:

#+begin_src julia :session :results output :exports both :eval never-export
FsMax = 66e6/1<<8
@printf "Fs <= %s Hz" cfmt("%'d", FsMax)
#+end_src

#+RESULTS:
: Fs <= 257,812 Hz

\(F_{I}\) is found by dividing the frequency of ~PLL4_MAIN_CLK~
(\(F_{M}\)) by the ~SAI1_CLK_PRED~ (\(D_{I_{1}}\)) and ~SAI1_CLK_PODF~
(\(D_{I_{2}}\)) fields of the ~CCM_CS1CDR~ register:

#+NAME: eq:sai1-freq1
\begin{equation}
F_{I} = \frac{F_{M}}{D_{I_{1}}D_{I_{2}}}
\end{equation}

\(F_{M}\) is found by dividing \(F_{P}\), the frequency of PLL4, by
~POST_DIV_SELECT~ (\(D_{p}\)) and ~AUDIO_DIV~ (\(D_{A}\)):

#+NAME: eq:pll4-main
\begin{equation}
F_{M} = \frac{F_{P}}{D_{p}D_{A}}
\end{equation}

\(F_{P}\) is the result of the multiplying the 24 MHz clock source
(\(F_{X}\)) by the PLL4 divider, \(D_{P}\):

#+NAME: eq:pll4-freq
\begin{equation}
F_{P} &= F_{X}D_{P}
\end{equation}

\(D_{P}\) is, in turn, the sum of the /PLL loop divider/, the
~DIV_SELECT~ field of ~CCM_ANALOG_PLL_AUDIO~ (\(D_{S}\)) with the
quotient of the /PLL fractional loop divider/ numerator
(~CCM_ANALOG_PLL_AUDIO_NUM~, \(D_{N}\)) and denominator
(~CCM_ANALOG_PLL_AUDIO_DENOM~, \(D_{D}\)) registers:

#+NAME: eq:pll4-divider
\begin{equation}
D_{P} = D_{S} + \frac{D_{N}}{D_{D}}
\end{equation}

The ~DIV_SELECT~ field has a valid range of \(27 \leq D_{S} \leq 54\), and
\(|D_{N}| < D_{D}\), so \(D_{P}\) describes (in theory) a number in the
following range:

\[
\left(26 + \frac{1}{D_{D}}\right) \leq D_{P} \leq \left(54 + \frac{D_{D}-1}{D_{D}}\right),
\]
with \(1 \leq D_{D} \leq 2^{30}\); effectively, \(26 < D_{P} < 55\). MCUXpresso Config
Tools, and the reference manual (see §13.3.2.2 and §14.6.1.3.4),
indicate, however, that the output frequency range for PLL4 is:

#+NAME: eq:pll4-range
\begin{equation}
650\,\text{MHz} \leq F_{P} \leq 1\,300\,\text{MHz},
\end{equation}

plus Config Tools contradicts the manual and states that \(D_{N}\) must
be a /positive/ 30 bit integer, so:

#+begin_src julia :session :results output :eval never-export :exports both
DpMin = 650e6/24e6
DpMax = 1300e6/24e6
@printf "%f <= PLL4 divider <= %f" DpMin DpMax
#+end_src

#+RESULTS:
: 27.083333 <= PLL4 divider <= 54.166667

\[
\left(27 + \frac{1}{12}_{}\right) \leq D_{P} \leq \left(54 + \frac{1}_{}{6}\right).
\]

Setting extreme (but valid) values for the divisors, we can find the
minimal valid \(F_{s}\):

#+begin_src julia :session :results output :eval never-export :exports both
# Maximal values for POST_DIV_SELECT and AUDIO_DIV
POST_DIV_SELECT = 4
AUDIO_DIV = 4
pll4Div = POST_DIV_SELECT * AUDIO_DIV
# Maximal values for SAI1_CLK_PRED and SAI1_CLK_PODF
SAI1_CLK_PRED = 8
SAI1_CLK_PODF = 64
sai1Div = SAI1_CLK_PRED * SAI1_CLK_PODF
# See earlier listing for DpMin
FsMin = (OSC_24M * DpMin)/(pll4Div*sai1Div*1<<8)
@printf "%s Hz <= Fs <= %s Hz" cfmt("%'d", FsMin) cfmt("%'d", FsMax)
#+end_src

#+RESULTS:
: 310 Hz <= Fs <= 257,812 Hz

** General Expression for the Sample Rate

Bringing all of the above together, the general expression for
achieving target sample rate \(\bar{F}_{s}\) is:

#+NAME: eq:general-fs
\begin{align}
\bar{F}_{s} &= \frac{F_{I}}{2^{8}} \nonumber \\
  &= \frac{F_{M}}{2^{8}D_{I_{1}}D_{I_{2}}} \nonumber \\
  &= \frac{F_{X}D_{P}}{2^{8}D_{p}D_{A}D_{I_{1}}D_{I_{2}}} \nonumber \\
  &= \frac{24\times10^{6}}{2^{8}}\frac{D_{S}+\frac{D_{N}}{D_{D}}}{D_{p}D_{A}D_{I_{1}}D_{I_{2}}}
\end{align}

This reckoning alone doesn't take into account certain constraints on
the system, however, such as eqref:eq:pll4-range and the frequency at
the input of \(D_{I_{2}}\) being less than 300 MHz. There are too many
free variables to solve for any one of them, so an iterative approach
is probably going to be necessary.

* Setting a different audio sample rate

Say we want to set a sample rate of \(F_{s} = 48\,\text{kHz}\). Our
target \({F_{I}}\), \(\bar{F}_{I}\), equals \(\bar{F}_{S}\cdot2^{8}\):

#+begin_src julia :session :results output :exports both :eval never-export
Fs = 48000
Fi = Fs * 1<<8
@printf "Target SAI1_CLK_ROOT: %s Hz" cfmt("%'d", Fi)
#+end_src

#+RESULTS:
: Target SAI1_CLK_ROOT: 12,288,000 Hz

Let's (strategically) ignore \(D_{p}\) and \(D_{A}\), which we'll leave set
to 1. Via the following iterative code we can find a selection of
register and field options:

#+begin_src julia :session :results output :eval never-export :exports both
let
    wordsize = 1 << 8
    # Adjustment to all figures so we can compare integer targets...
    multiplier = OSC_24M/wordsize
    # Frequency limitaitions
    pll4Min = 650e6
    pll4Max = 1300e6
    sai1PostMaxInputFreq = 300e6

    @printf "Target sample rate:   %s Hz\n" cfmt("%'d", Fs)
    @printf "Target SAI1_CLK_ROOT: %s Hz\n\n" cfmt("%'d", Fi)

    for div in 27:54
        num = 0           # Initial numerator
        denom = 10        # Start with tenths
        sai1Pre = 1       # Initial SAI1 pre-divider
        sai1Post = 1      # and post-divider
        divExact = -1.0

        while OSC_24M * (div + num / denom) < pll4Min
            num +=1
        end            

        pllFreq = OSC_24M * (div + num / denom)
        # num = Int(num * 1e5)
        # denom = Int(denom * 1e5)

        # Adjust first sai1 divider to get a valid input to the second
        # divider
        while pllFreq / sai1Pre > sai1PostMaxInputFreq && sai1Pre < 8
            sai1Pre += 1
        end

        while floor(divExact) != div
            # Adjust second sai1 divider to get a value below the target
            while multiplier * div / (sai1Pre * sai1Post) > Fs && sai1Post < 64
                sai1Post += 1
            end

            # If there's more than one multiplier between us and success,
            # try a different PRED/PODF combination.
            if (Fs * sai1Pre * sai1Post - div * multiplier > multiplier
                   && sai1Pre < 8)
                sai1Pre += 1
                sai1Post = 1
                continue
            end

            divExact = Fs * wordsize * sai1Pre * sai1Post / OSC_24M

            # If fractional and integer div don't have the same
            # integer part, something's wrong; try a different SAI1
            # predivider.
            if floor(divExact) != div
                if sai1Pre < 8
                    sai1Pre += 1
                    sai1Post = 1
                    continue
                else
                    break
                end
            end

            num = floor(divExact * denom - div * denom)

            # Scale denom and recalculate num until the clock
            # calculation matches the target; if it doesn't by the
            # time we're dividing by a million, it (probably) isn't
            # going to happen.
            while (floor(OSC_24M * (div + num / denom) / (sai1Pre * sai1Post)) != Fi
                   && denom <= 1e6)
                denom *= 10
                num = floor(divExact * denom - div * denom)
            end
            
            if denom <= 1e6
                # Found a match
                @printf "24 MHz * (%d + %d/%d)" div num denom
                @printf "/(1 * 1 * %d * %d)" sai1Pre sai1Post
                @printf " = %f\n" 24e6 * (div + num / denom) / (sai1Pre * sai1Post)
            else
                break
            end
        end
    end
end
#+end_src
